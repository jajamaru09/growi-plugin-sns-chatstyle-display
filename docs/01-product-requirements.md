# プロダクト要求定義書

## 1. プロダクトビジョンと目的

### ビジョン
Growi上のMarkdownページで、LINEやSlackのようなSNS風チャットスタイルの会話表現を簡単に記述・表示できるようにする。

### 目的
- 技術ドキュメントにおける対話形式の説明をわかりやすく表現する
- FAQ、チュートリアル、ロールプレイ形式のコンテンツを直感的に記述・閲覧できるようにする
- Growiの標準Markdown記法を拡張し、プラグインとして配布可能な形で提供する

## 2. ターゲットユーザーと課題・ニーズ

### ターゲットユーザー
- Growiを利用しているチーム・組織のWiki編集者
- 対話形式でドキュメントを作成したいテクニカルライター
- FAQ形式のナレッジベースを構築したい管理者

### 課題
- 現在のGrowiには対話形式の表現手段がなく、テキストベースの引用やリストで代用するしかない
- 会話の流れが視覚的にわかりにくい

### ニーズ
- Markdownで簡単に書ける記法（既存のコールアウト記法に近い形）
- 左右に分かれたチャットバブル表示（左＝相手、右＝自分）
- アイコン画像と話者名の表示
- 表情差分や複数キャラクターへの対応

## 3. 主要な機能一覧

| # | 機能 | 概要 |
|---|------|------|
| F-01 | 会話ブロック記法 | `::::chat-style` で会話モードの開始・終了を制御するラッパーコンテナ |
| F-02 | チャットバブル記法 | 会話ブロック内で `:::speaker_expression` 形式の吹き出しを記述 |
| F-03 | 左右配置レンダリング | 話者定義の配置設定に従い左右にバブルを配置 |
| F-04 | アイコン表示 | 話者定義ページのアイコン画像を吹き出しの横に表示 |
| F-05 | 話者名表示 | 吹き出しの上部に話者名を表示 |
| F-06 | 表情差分対応 | `_smiling`, `_default` 等で同一話者の表情を切り替え可能（省略時は `_default`） |
| F-07 | Markdown内テキスト | 吹き出し内でインラインMarkdown（太字・リンク等）を利用可能 |
| F-08 | 話者定義ページ | `/chat-style-icons` ページでアイコン・配置をテーブル形式で管理 |

## 4. 成功の定義

- Growiにプラグインとしてインストールでき、再起動後に機能する
- 専用記法でMarkdownを書くと、SNS風チャットUIがレンダリングされる
- 編集画面のプレビューでもリアルタイムに反映される
- 既存のMarkdownコンテンツに影響を与えない
- `/chat-style-icons` ページでアイコン定義を変更すると、全ページに反映される

## 5. ビジネス要件

- OSSとして公開可能な形で開発する
- Growiプラグインの標準仕様に準拠する
- 外部依存を最小限に抑え、メンテナンスコストを低く保つ

## 6. ユーザーストーリー

### US-01: 通常記事の中に会話形式を挿入する
**As a** Wiki編集者
**I want to** 通常のMarkdown記事の途中で会話形式ブロックを開始・終了できる
**So that** 解説文と対話形式を自由に組み合わせたドキュメントを作れる

### US-02: キャラクターの使い分け
**As a** テクニカルライター
**I want to** 複数の話者（ボット、ユーザー、管理者など）を使い分ける
**So that** 役割の違いが視覚的に明確になる

### US-03: 表情の切り替え
**As a** コンテンツ作成者
**I want to** 同じ話者でも場面に応じて表情を変える
**So that** 対話にニュアンスや感情を加えられる

### US-04: アイコンの管理
**As a** Wiki管理者
**I want to** 専用ページで話者のアイコンや配置を一元管理する
**So that** チーム全体で統一されたキャラクター表現を使える

## 7. 受け入れ条件

- [ ] `::::chat-style` ブロック内でのみチャット記法が有効になる
- [ ] `::::chat-style` ブロック外の `:::speaker` 記法はプラグインの処理対象にならない
- [ ] `:::speaker_expression` 記法で吹き出しがレンダリングされる
- [ ] `:::speaker`（expression省略）で `_default` のアイコンが使用される
- [ ] 1ページ内に複数の `::::chat-style` ブロックを配置できる
- [ ] `/chat-style-icons` ページのテーブル定義に従い左右配置・アイコンが表示される
- [ ] 未定義の話者が使用された場合、エラーメッセージが表示される
- [ ] 話者名が吹き出し上部に表示される
- [ ] 吹き出し内でインラインMarkdown（太字、リンク、コード）が機能する
- [ ] 既存のMarkdownコンテンツに影響がない
- [ ] Growiの標準プラグインインストール手順で導入できる

## 8. 機能要件

### 8.1 記法仕様

#### 会話ブロック（ラッパー）

`::::chat-style`（コロン4つ）で会話モードの開始・終了を制御する。
このブロックの内側でのみ、話者記法（`:::speaker`）がチャットバブルとして処理される。

```markdown
# 通常のMarkdown

ここは普通のドキュメントです。以下から会話形式になります。

::::chat-style
:::claude-bot_smiling
会話内容をここに書きます。
**太字**や`コード`も使えます。
:::

:::me
expressionを省略するとdefaultが使われる
:::
::::

ここからまた通常のMarkdownに戻ります。
```

- `::::chat-style` — 会話ブロックの開始（コロン4つ）
- `::::` — 会話ブロックの終了（コロン4つ）
- ブロック外の `:::` 記法はプラグインの処理対象外（他のプラグインやGrowi標準機能に影響しない）
- 1ページ内に複数の会話ブロックを配置可能

#### 話者記法（会話ブロック内）

- `:::話者名_表情` — 話者名と表情をアンダースコアで区切る
- `:::話者名` — 表情を省略した場合は `default` を使用
- 話者名・表情は `/chat-style-icons` ページの定義と一致する必要がある
- 未定義の話者が使われた場合はエラー表示とする

### 8.2 話者定義ページ (`/chat-style-icons`)

Growiの通常ページとしてテーブル形式で話者情報を管理する。

```markdown
# チャットスタイル アイコン定義

| 話者 | expression | アイコン | 配置 |
|------|-----------|---------|------|
| claude-bot | default | ![](/attachment/xxx) | left |
| claude-bot | smiling | ![](/attachment/yyy) | left |
| me | default | ![](/attachment/zzz) | right |
| admin | default | ![](/attachment/aaa) | left |
```

- **話者**: 記法で使用する話者名
- **expression**: 表情の識別子（`default` は必須）
- **アイコン**: Markdown画像記法で指定（Growi添付ファイルや外部URL）
- **配置**: `left`（左＝相手側）または `right`（右＝自分側）

### 8.3 レンダリング仕様
- 左配置: アイコン（左） → 話者名 → 吹き出し（左寄せ）
- 右配置: 吹き出し（右寄せ） → 話者名 → アイコン（右）
- 吹き出しは角丸の矩形で、話者側に三角形のポインターを持つ
- 背景色は左右で異なる（左: 白/グレー系、右: 緑/青系）

### 8.4 エラーハンドリング
- 未定義の話者が使用された場合: 赤色のエラーメッセージを表示（例: `[chat-style error] 話者 "unknown" は /chat-style-icons に定義されていません`）
- `/chat-style-icons` ページが存在しない場合: エラーメッセージを表示

## 9. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | ページ読み込み時間への影響が100ms以内 |
| ブラウザ対応 | Chrome, Firefox, Edge の最新2バージョン |
| アクセシビリティ | 適切なARIAラベル、スクリーンリーダー対応 |
| レスポンシブ | モバイル表示でも崩れないレイアウト |
| 互換性 | Growi v7.x 以降に対応 |
